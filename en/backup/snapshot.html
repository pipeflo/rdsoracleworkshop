Snapshot Restore

To restore a new RDS Oracle instance from a Snapshot, please follow the next steps:

  1. On the RDS Console page click “Snapshots” on the left menu.
  2. Select the snapshot of your Database which you plan to use for the Restore, then click the Actions button and then click Restore Snapshot
  3. In the “Restore DB Instance” page, you can select the options for the new instance that is going to be restored from the selected snapshot:
    • On the “Instance specifications” section, change the DB Instance Class to db.t3.small.
    • On the Settings section, enter the DB Instance Identifier name, for example myrestoredoracledb
    • On the Network & Security section, select the option Choose existing VPC security groups and then select the Security Group created on the “Deployment” chapter, in my case oracledb and remove the Default security group.
    • On the Database options enter MRDB as the Database Name.
    • Leave the other field as they are and click Restore DB Instance at the bottom of the page.
  4. Now please wait until the restore is finished; This will create a new RDS Instance from the Snapshot selected on the second step. You can see the status in the Databases list.
