Parameter Groups

Creating the Parameter Group

  1. On the RDS home page, select “Parameter Groups” on the left-hand side and then click on the Create parameter group button at the top right corner.
  2. On the Create Parameter group provide input as below and then click Create:
    • Parameter group family - choose “oracle-ee-19”
    • Group name – custom-oracle19-parameter-group
    • Description - Custom parameter group for Oracle 19

Modifying the Parameter in the parameter Group

  1. On the Parameter groups table, select the parameter group created (custom-oracle19-parameter-group), click the Parameter group actions button and then click on Edit.
  2. Under the “filter parameters” type job_queue_process, change the job_queue_process value from 50 to 100 and then click the Save Changes button.
    • In case you change make several changes to the custom parameter group, you can see preview them before saving by clicking the Preview changes button. Now that the changes are saved, you are going to associate the new parameter group to your RDS Oracle instance.

Associating new Parameter Group to your Instance

  1. Find the RDS Oracle created instance under the Databases list and click on the DB Instance name.

  2. On the RDS Instance details, click on the Modify button.

  3. On the next page under “Database Options” change the DB Parameter Group from the dropdown to the new Parameter Group “custom-oracle19-parameter-group” created in the previous step, leave all other fields as they are and then click Continue at the bottom of the page.

  4. Now select Apply Immediately on the “Scheduling of modifications” section and then click Modify DB Instance.

  5. Once the modify instance task is completed, this can be verified by looking at the status column for your Database instance on the Databases page. When the status column shows “Available” it means the modification to the instance is completed.

  6. We need to reboot the Instance manually to apply the new parameter group to the Instance. This can be verified on the Instance details page under the parameter Group section which will show “Pending-reboot”

  7. Reboot the instance manually, choose your Instance from the RDS home page and under Instance actions choose “Reboot”.

  8. Click Reboot

  9. Verify the status of the instance on the RDS home page, once the status of the instance becomes available, verify the value of the parameter “job_queue_process” to make sure it has changed to 100 by connecting to your RDS Instance using SQL*Plus and running the following command:

    show parameter job_queue;

Congratulations!! You have created and applied a Parameter Group to you RDS Oracle Instance.

Parameter Group behaviour

  • When you change a dynamic parameter and save the DB parameter group, the change is applied immediately regardless of the Apply Immediately setting. When you change a static parameter and save the DB parameter group, the parameter change will take effect after you manually reboot the DB instance.

  • When you change the DB parameter group associated with a DB instance, you must manually reboot the instance before the new DB parameter group is used by the DB instance.

  • The value for a DB parameter can be specified as an integer or as an integer expression built from formulas, variables, functions, and operators. Functions can include a mathematical log expression

  • Improperly setting parameters in a DB parameter group can have unintended adverse effects, including degraded performance and system instability. Always exercise caution when modifying database parameters and back up your data before modifying a DB parameter group
