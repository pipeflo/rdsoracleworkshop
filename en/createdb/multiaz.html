Automatic Failover with Multi-AZ

In this section you will see how the Multi AZ feature selected when you created your RDS for Oracle instance, provides automatic failover in case there is any failure with the primary instance.

  1. First verify that the Multi-AZ feature is turned on, for this, go to the RDS Databases list and click on the Oracle Instance you created on the first chapter of the workshop, now, click on the Configuration tab and then scroll down and check that the Multi AZ option has the Yes value.

    Note: In case you did not turned on the Multi AZ feature you can follow the next steps to turn it on.

    • Click on the Modify button at the top of the page.
    • Here you can modify some of the features of your RDS Instance, please look for the Multi-AZ deployment option and change the value to Yes, then go to the bottom of the page and click on Continue
    • Now select Apply Immediately on the “Scheduling of modifications” section and then click Modify DB Instance, this will excecute the modifications as soon as possible and will not wait unti the next configured maintenance window.
    • Now please wait until the Modification finishes. You can check the status on the Databases list.

  2. Now, you are going to simulate the AZ Failover on the RDS Oracle instance.

    RDS Oracle does provide customer’s an option to simulate the AZ failure and High Availability by offering the option to reboot the RDS Oracle Instance with the failover option. This option will initiate AZ level failover for the Instance, the instance on the secondary AZ will become primary and the instance on the primary become secondary.

    • Verify the primary and secondary AZ for the RDS Oracle Database Instance. Choose your RDS Oracle Instance from the RDS databases list page by clicking Databases and after selecting your instance look for the “Availability zone” value on the “Networking” column.
    • To initiate AZ failover, click on the Actions button on the top right and then click on Reboot
    • Choose the checkbox for Reboot with Failover and click on the Reboot to initiate the failover.
    • Once the RDS Oracle Database Instance is rebooted and status shows available, check for the value of the Availability Zone and it should now display a different one from the previous one.

Summary

Congratulations!! you have finished the first lab of this RDS for Oracle Workshop, where you created a new RDS Oracle Instance, connected to it and then simulated a failover using the Multi AZ future of RDS. Here is a list of the benefits you have tested using RDS:

Benefits

  • Replication to a standby replica is synchronous which is highly durable.
  • When a problem is detected on the primary instance, it will automatically failover to the standby in the following conditions:
    • The primary DB instance fails
    • An Availability Zone outage
    • The DB instance server type is changed
    • The operating system of the DB instance is undergoing software patching.
    • A manual failover of the DB instance was initiated using Reboot with failover.
  • The endpoint of the DB instance remains the same after a failover, the application can resume database operations without manual intervention.
  • If a failure occurs, your availability impact is limited to the time that the automatic failover takes to complete. This helps to achieve increased availability.
  • It reduces the impact of maintenance. RDS performs maintenance on the standby first, promotes the standby to primary master, and then performs maintenance on the old master which is now a standby replica.
  • To prevent any negative impact of the backup process on performance, Amazon RDS creates a backup from the standby replica.
