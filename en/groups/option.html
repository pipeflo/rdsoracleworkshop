Option Groups

Creating Option Groups

  1. On the RDS home page, select “Option Groups” on the left-hand side menu and then click the Create group button.
  2. On the “Create option group” page provide the following inputs and click Create:
    • Name – User defined custom name (ora19ccustomgroup)
    • Description – option group for SQLT option
    • Engine – select oracle-ee
    • Major engine version – select 19

Adding options to the Option Groups

  1. On the RDS home page, click “Option Groups“ on the left-hand side menu, select the option group created in the previous step (ora19ccustomgroup) and click on the “Add option” button

    • In this option Group, we are adding “SQLT” option, query the Database to verify it is not installed currently by runnning the following query using SQL*Plus 
      SELECT sqltxplain.sqlt$a.get_param('tool_version') sqlt_version FROM DUAL;

  2. On the Option Details page, provide following inputs and click Add option:

    • For the option, choose the option “SQLT” from the dropdown
    • For the version field leave the default value
    • For the License_pack field provide “T”
    • Apply Immediately field provide “Yes”

Associating Option Group to your Instance

  1. Find the RDS Oracle created instance under the Databases list and click on the DB Instance name.
  2. On the RDS Instance details, click on the Modify button.
  3. On the next page under “Database Options” change the Option group from the dropdown to the new Option Group “ora19ccustomgroup” created in the previous step, leave all other fields as they are and then click Continue at the bottom of the page.
  4. Now select Apply Immediately on the “Scheduling of modifications” section and then click Modify DB Instance.
  5. Once the status of the database is “Available”, verify that SQLT is installed on the database by running the following query using SQL*plus: 
    SELECT sqltxplain.sqlt$a.get_param('tool_version') sqlt_version FROM DUAL;

Congratulations!! You have created and applied an Option Group to you RDS Oracle Instance.

There are two types of Option Groups:

  1. Permanent Option Group:

    Means that option can never be removed from the  option group. Also, if an option group attached to an instance contains permanent options, you can only replace the option group with another option with those permanent options.

    Permanent options, such as the TDE option for Oracle Advanced Security TDE, can never be removed from an option group, and the option group cannot be disassociated from the DB instance.

  2. Persistent Option Group:

    Means the option can only be removed if the option group is not attached to any DB instance
