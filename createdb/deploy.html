Create an RDS Oracle Instance

  1. Sign in to the AWS Management Console and open the Amazon RDS console at console.aws.amazon.com/rds/

  2. On the top right corner of the Amazon RDS console, choose the AWS Region in which you want to create the DB instance. All participants are required to choose the AWS region provided by the Instructor.

  3. Choose “Create Database”

  4. In the Create Database screen select the following options:

  • Database Creation Method select Standard Create

  • Engine Options click Oracle and then select Oracle 19.0.0.0.ru-2019.rur-2019-07.r1.

  • On the templates section, select Dev/Test

  • On settings, insert the DB instance name (this will be the name that will identify your RDS instance on the databases list in th RDS console) and on the credentials settings leave admin as the master user and insert a password (this will be the user credentials that you will use later to connect to the Oracle instance).

  • On instance size, select Burstable classess and then select the db.t3.small instance.

  • On storage, leave the default options, here you can see that the Storage autoscaling option is selected, this will reduce your administrative tasks by automatically scaling the storage of your rds instance.

  • On the Availability & Durability secction, select the option “Create a standby instance (recommended for production usage)", this will create a MultiAZ deployment which will deploy a secondary instance on a different Availability Zone, and Amazon RDS will perform automatic failover in case there is a filure on the primary instance

  • On Connectivity, select the Default VPC and then expand the Additional connectivity configuration section, and here, select Yes for the Publicly accessible option, then select Create new for the VPC Security Group and enter Oracle for the VPC Security group name and leave the default port 1521, this will create a new security group on your default vpc, with the name Oracle, the port 1521 for the port range and you current public ip as the source

  • On Database authentication leave the default option (Password authentication)

  • To finish, expand the Additional Configuration section, enter MFDB as the “Initial database name” then leave all the other options with the default selections and finally click on Create Database.

Now your RDS for Oracle Instance is deploying, this will take a few minutes to be ready. Once the status on the databases list changes from Creating to Available you can continue with the next section.
