Configure Optimized CPU Feature

Amazon RDS DB instance classes support Intel Hyper-Threading Technology, which enables multiple threads to run concurrently on a single Intel Xeon CPU core. Each thread is represented as a virtual CPU (vCPU) on the DB instance. A DB instance has a default number of CPU cores, which varies according to DB instance type. For example, a db.r4.xlarge DB instance type has two CPU cores and two threads per core by default—four vCPUs in total.

Optimize CPU feature can be used to optimize your DB instance for specific workloads or business needs:

  • Number of CPU cores – You can customize the number of CPU cores for the DB instance. You might do this to potentially optimize the licensing costs of your software with a DB instance that has sufficient amounts of RAM for memory-intensive workloads but fewer CPU cores.
  • Threads per core – You can disable Intel Hyper-Threading Technology by specifying a single thread per CPU core. You might do this for certain workloads, such as high-performance computing (HPC) workloads.

You can control the number of CPU cores and threads for each core separately. You can set one or both in a request. After a setting is associated with a DB instance, the setting persists until you change it.

You can configure the number of CPU cores and threads per core for the DB instance class when you perform the following operations:

  • Creating a DB instance
  • Modifying a DB instance
  • Restoring a DB instance from a snapshot
  • Restoring a DB instance to a point in time

Verifying the current DB processor configuration

  1. Find the RDS Oracle created instance under the Databases list and click on the DB Instance name.
  2. On the Instance details page click on the Configuration tab, on the Instance class section note the number of vCPUs currently configured for your DB instance.
  3. Query the vCPUs from the Database:
    • Login to your Database instance using the SQL*PLUS or any other client tools, below you can find the commands and screen shot (don’t forget to change wout user, hostname, SID and to enter your password). 
      sqlplus '<username>@(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=<Hostname>)(PORT=1521))(CONNECT_DATA=(SID=<SID>)))'

    • Query the “cpu_count” by entering the following query: 
      show parameter cpu_count;

Configuring Optimize CPU by Modifying the Instance

  1. Return to the RDS Console, and click on the Modify button at the top of the page.
  2. Click on Additional configuration at the bottom of the “Instance specifications” section, and select the Checkbox to Override default values, then change the Threads per core value from 2 to 1, leave all other fields as they are and then click Continue at the bottom of the page.
  3. Now select Apply Immediately on the “Scheduling of modifications” section and then click Modify DB Instance, this will excecute the modifications as soon as possible and will not wait unti the next configured maintenance window.
  4. Wait a few minutes for the change to be applied, and then connect again to your RDS Oracle instance and run the “cpu_count” query to see the changes.
