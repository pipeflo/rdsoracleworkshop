Create an RDS Oracle Instance

  1. Sign in to the AWS Management Console and open the Amazon RDS console at console.aws.amazon.com/rds/

  2. On the top right corner of the Amazon RDS console, choose the AWS Region in which you want to create the DB instance. All participants are required to choose the AWS region provided by the Instructor.

  3. Choose “Create Database”

  4. In the Create Database screen select the following options:

    • Database Creation Method select Standard Create
    • Engine Options click Oracle and then select Oracle 19.0.0.0.ru-2019.rur-2019-07.r1.
    • On the templates section, select Dev/Test
    • On settings, insert the DB instance name (this will be the name that will identify your RDS instance on the RDS dashboard) and on the credentials settings leave admin as the master user and insert a password (this will be the user credentials that you will use later to connect to the Oracle instance).
    • On instance size, select Burstable classess and then select the db.t3.small instance.
    • On storage, leave the default options, here you can see that the Storage autoscaling option is selected, this will reduce your administrative tasks by automatically scaling the storage of your rds instance.
    • On the Availability & Durability secction, select the option “Create a standby instance (recommended for production usage)", this will create a MultiAZ deployment which will deploy a secondary instance on a different Availability Zone, and Amazon RDS will perform automatic failover in case there is a filure on the primary instance
