View RDS Oracle logs on CloudWatch

View Oracle logs on Cloudwatch

  1. On the AWS console home page type CloudWatch in the search box and click on CloudWatch.

  2. On the CloudWatch home page click on “Logs” on the left-hand side.

  3. If you get a welcome message, please click on the Geting started button and then enter /aws/rds/instance/ on the filter tab and hit enter.

View Performance Insights

  1. We have already enabled the Performance insights for this instance with the previous modification. Accessing Performance Insights require “AmazonRDSFullAccess” permissions from AWS Identity and Access Management (IAM).

  2. To access the Performance Insight Dashboard for specific RDS Database Instance, go to the Databases list page and select the desired Instance name and under the “current activity” column you will see the average number of sessions in last 5 minutes. Click on the “sessions” link and it will take you to the Performance Insights Dashboard for that specific Instance. Another way to access Performance Insights, go to the RDS homepage and click on the Performance Insights tab on the left-hand side On the next screen type the instance name you want to monitor (in this your RDS Oracle Database Instance) and click on the Instance name.

  3. With the Performance Insights dashboard, you can visualize the database load and filter the load by waits, SQL statements, hosts, or users. The central metric for Performance Insights is DB Load, which represents the average number of active sessions for the DB engine. An active session is a connection that has submitted work to the DB engine and is waiting for a response from it. The Dashboard is divided into 3 sections Counter Metrics, Database Load and a Top activity table that shows what is contributing to database load for the time interval on the load chart.

    • Counter Metrics

      With counter metrics, you can customize the Performance Insights dashboard to include up to 10 additional graphs. These graphs that show a selection of dozens of operating system and database performance metrics. This information can be correlated with database load to help identify and analyze performance problems. You can customize tue metrics displayed by clicking on the settings icon () at the top right corner and then selectiong the metrics you want to see.

    • Database Load Here you can see the Load generated to the Instance and filter by Waits, SQL, Hosts and Users.

    • Top activity table

Congratulations!! You have learned how to see the RDS Logs on Cloudwatch and how to use the Performance Insights to analyze database load.
